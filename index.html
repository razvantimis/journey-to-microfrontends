Evolution of software architecture

- we are going to talk about the evolution of software architecture,from monolithic to micro frontends / services - we want to see how we got to micro frontends and why
- Avalability - the system is available to the users 24/7 - Reliability - the system is reliable and can recover from failures - Testability - the system is testable - Scalability - the system can handle the load and can scale horizontally - Security - the system is secure - Agility - the system is agile and can be changed fast - Fault tolerance - the system can handle failures - Elasticity - the system can scale up and down based on the load - Recoverability - the system can recover from failures - Performance - the system is performant - Deployability - the system can be deployed fast - Learnability - the system is easy to learn

Monolithic architecture

- Most of companies have a monolithic architecture. - This means we one singple application and the frontend and backend are included in the same application.

1.1 Independent deployments

Each team can deploy their part of the application independently. This is a huge win for teams that operate independently and want to move fast.

What is a micro frontend?

1.1 similiar to microservices 1.2 Like youtube embedded video or an iframe from other website 1.3 The term Micro Frontends first came up in ThoughtWorks Technology Radar at the end of 2016. It extends the concepts of micro services to the frontend world.
